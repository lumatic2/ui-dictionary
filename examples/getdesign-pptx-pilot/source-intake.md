# Source intake — Linear analysis

## Provenance

- Source: `getdesign add linear.app --out tmp/getdesign-intake/linear-pilot-20260721.DESIGN.md`
- Retrieved: 2026-07-21
- Nature: getdesign's independent analysis of publicly observable patterns, not an official Linear design system.
- Intended use: a per-project starting point for a new Askewly Design introduction; not a brand reproduction.

## Retained principles

- Near-black canvas with a restrained four-step surface ladder.
- One lavender accent reserved for focus and the primary action.
- Dense, quiet product-documentation rhythm; hairline borders before shadows.
- Tight display hierarchy and mono labels only where they communicate system metadata.

## Deliberately excluded

- Linear name, logo, product screenshots, customer marks, and marketing copy.
- `Linear Display`, `Linear Text`, and `Linear Mono`; this pilot uses the system-safe `Inter, Noto Sans KR, Arial, sans-serif` stack.
- Linear's exact color values, component names, and pill-heavy controls.
- Any claim that this is Linear's official system.

## Project decisions

- Project subject: Askewly Design, a design system for people and coding agents.
- Deliverables: one web page and one editable 16:9 six-slide deck.
- Design owner: the adjacent `DESIGN.md`. Askewly recipes and checks guide quality, but do not replace these project tokens.
