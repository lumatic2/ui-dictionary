# Design Brief — 착수 인터뷰 계약

Date: 2026-07-18
지위: 정본. 진입 프로토콜(entry-protocol.md)의 브리프 단계가 이 문서를 가리킨다.

시니어 디자이너는 의뢰를 받으면 바로 그리지 않는다 — 브리프를 받는다. 이 문서는 코딩 에이전트가 디자인 작업 착수 전에 사용자 의도를 수집하는 계약이다. 목적은 두 가지: ① 추정을 질문으로 대체한다 ② 답변을 프로젝트의 `DESIGN.md`로 저장해 **프로젝트가 룩 소유권을 갖게 만든다** — 이후 작업은 재질문 없이 그 파일에서 파생한다.

## 1. 규모 게이트 — 언제 인터뷰하는가

| 상황 | 판정 |
|---|---|
| 프로젝트에 `DESIGN.md`(또는 토큰 SSOT)가 **이미 있음** | **인터뷰 없음** — 그 파일에서 파생. 파일이 답하지 못하는 새 결정만 개별 질문 |
| 새 페이지·사이트·화면·제품 (룩이 처음 정해지는 시점) | **풀 브리프** — §2의 7도메인 인터뷰 → §3 DESIGN.md 저장 |
| 컴포넌트 1개·기존 UI 수정·폴리싱 | **생략** — 기존 스타일에서 파생. DESIGN.md도 토큰도 없으면 최소 질문 1개(컬러 계열)만 |
| 사용자가 "인터뷰 없이", "알아서" 명시 | 생략 — 합리적 기본값 + 보고에 추정 목록 명시 |

오발동 금지: 버튼 하나 고치는 작업에 질문 10개를 던지는 것은 게이트 실패다.

## 2. 질문 카탈로그 — 7도메인

각 도메인은 **선택지 2~4개 + 추천 1개(근거 한 줄)**로 제시한다. 전부 주관식으로 묻는 것은 금지(사용자 피로). Claude는 AskUserQuestion으로 묶어서(한 번에 최대 4도메인), Codex는 평문 일괄 질문으로. 사용자가 답을 건너뛰면 추천값 채택으로 간주하고 기록한다.

1. **정체성·톤** — 누구를 위한 무엇인가. 톤 후보: 따뜻함/신뢰·정제/미니멀/힙·에너지. 참고하고 싶은 사이트·브랜드가 있는가(있으면 관찰 대상, look 복사 금지).
2. **룩 소유권** — 기존 브랜드 자산(로고·색·서체)이 있는가. 있으면 그 값에서 파생, 없으면 3~4로.
3. **컬러** — 베이스(웜 페이퍼/쿨 그레이/순수 다크) + 액센트 1색 계열(예: 레드 계열/블루/그린/바이올렛/앰버). 액센트는 1색이 기본 — 2색 이상은 사용자가 명시할 때만.
4. **타이포그래피** — 서체 느낌: 시스템 산세리프(기본)/개성 산세리프/세리프 혼용. 디스플레이(제목) 크기 성향: 크고 압축 vs 절제.
5. **구조** — 필요한 블록: 헤더(내비 항목 수), 히어로(카피 방향), 본문 섹션(무엇을 보여줄지), 푸터(연락처/링크/법적 고지). 화면이 여러 개면 목록 확정.
6. **인터랙션 수위** — 담백(전환만)/표준(hover·등장 모션)/풍부(스크롤 연출·커서 반응 — 티어 지도 판정 + 실험적 터치는 수동 원칙 적용).
7. **콘텐츠·범위** — 실데이터(메뉴·가격·주소 등)를 줄 수 있는가 vs 더미로 시작. 다크 모드 필요 여부(기본: 제공). 모바일 우선 여부.

## 3. DESIGN.md 저장 계약

- 풀 브리프의 답변은 **작업 프로젝트 루트 `DESIGN.md`**로 저장한다 — 포맷은 askewly `templates/DESIGN.md.tmpl`(YAML frontmatter 토큰 + 본문 rationale)을 따르고, 답변에서 파생한 실제 토큰 값(베이스·액센트·서체·radius·간격)을 채운다.
- rationale 본문에 브리프 요약(톤·구조·인터랙션 수위·콘텐츠 출처)과 **추천값 채택 항목**(사용자가 명시하지 않아 기본값으로 간 것)을 구분해 기록한다 — 다음 세션이 "결정"과 "추정"을 구분할 수 있어야 한다.
- 이 파일이 생기는 순간 프로젝트는 룩 소유권을 가진다: 이후 모든 디자인 작업은 entry-protocol 토큰 판정에 따라 askewly 토큰이 아니라 이 파일에서 파생한다.

## 4. 대화 불가 환경(headless·자동화) 폴백

AskUserQuestion이 불가능한 환경에서는 인터뷰를 시도하다 멈추지 말 것: 규모 게이트만 적용하고, 풀 브리프 상황이면 7도메인 전부를 **추천 기본값으로 진행**하되 산출물 보고에 "브리프 미실시 — 추정 기본값 목록"을 반드시 명시한다. 사용자가 나중에 DESIGN.md의 추정 항목을 확정하면 그 값이 정본이 된다.
