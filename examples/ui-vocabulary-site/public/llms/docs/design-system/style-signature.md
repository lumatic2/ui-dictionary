# Askewly Style Signature

Status: canonical, user-approved 2026-07-17 (인터뷰 확정 — draft 역산 초안을 사용자 응답으로 재구성)

Audience: coding agents. 디자인 작업 결과물이 "유성 스타일"인지 판정하는 기준.

## 철학 (2026-07-17 사용자 확정)

**특정 스타일을 고정하지 않는다.** 팔레트, 그림자, radius, 밀도 같은 구체 값은 프로젝트가 소유한다(각 프로젝트의 DESIGN.md·토큰 SSOT). 프로젝트를 가로지르는 시그니처는 두 가지뿐이다:

1. **운용 원칙** — 값이 아니라 값을 다루는 방식
2. **명확한 비선호** — 위반하면 그 자체로 FAIL인 안티패턴

`principles.md`(보편 품질)와의 역할 분리는 유지한다. ui-dictionary 자체의 구체 규칙(white 시스템, hue 270 회색, pill 금지, 무그림자, 작은 radius 등)은 이 레포의 `DESIGN.md`가 소유하는 **프로젝트 시그니처**이지 전역 시그니처가 아니다.

## 운용 원칙 (프로젝트 무관 공통)

1. **토큰 파생** — 모든 색·치수는 프로젝트 토큰에서 파생한다. 프로젝트에 DESIGN.md/토큰 SSOT가 있으면 그것이 우선이고, 없으면 askewly `tokens.css`가 기본값이다. 발명 리터럴 0.
2. **액센트는 신호** — 강조색은 primary action·선택 상태·상태 표시 같은 신호로 소면적만 쓴다. 팔레트가 무엇이든 화면을 도포하지 않는다. 신호의 최소 단위는 글리프 1점까지 내려갈 수 있다 — 첫 화면의 브랜드 순간은 색 면적이 아니라 정확히 놓인 한 점으로도 성립한다 (사용자 승인 2026-07-17, taste ledger T-8).
3. **절제된 계층** — 뷰포트당 초점 1개. 계층은 간격·타이포·보더·표면 변화로 먼저 만들고, 경쟁하는 강조를 두지 않는다.
4. **상태 완비** — hover/focus/active/disabled/empty/error가 해당되는 만큼 존재하고, 라이트/다크 모두 성립한다.
5. **실험적 터치는 수동** — 의도적 비대칭·겹침·장식적 실험은 에이전트가 능동적으로 시도하지 않는다. 사람의 마감 터치이거나 명시적 요청이 있을 때만.

## 비선호 (위반 = FAIL — 2026-07-17 사용자 지목)

1. **좌측 컬러 라인 카드** — 카드 좌측면에 세로 액센트 라인을 붙이는 관습적 "알림/일정 카드" 패턴 (사례: 다크 카드 + 주황 좌측 보더 + 컬러 라벨 + 아이콘 칩 조합). AI 결과물의 대표적 관성 패턴.
2. **한글 단어 중간 잘림** — 한글 텍스트에 `word-break: keep-all` 미적용으로 단어가 줄바꿈에서 조각나는 것.
3. **어색한 문장 줄바꿈** — 제목·카피가 의미 단위를 무시하고 꺾이는 것. 제목류는 `text-wrap: balance`(또는 `pretty`), 필요 시 수동 개행 검토.
4. **이모지 아이콘** — UI 요소(버튼·라벨·상태·리스트 마커)에 이모지를 아이콘 대용으로 쓰는 것. 아이콘은 inline SVG.
5. **대충 그린 CSS** — 관측 가능한 형태: 정렬 어긋남(베이스라인·수직 중앙 불일치), 근거 없는 매직넘버 간격, 상태 없는 인터랙션 요소, 기본 브라우저 스타일 노출(비스타일 버튼·파란 링크), 토큰 무시 인라인 값.

## 판정 방식

점수제 없음. **운용 원칙 5개 전부 충족 + 비선호 0건 = 통과.** 하나라도 걸리면 해당 항목을 명시하고 고친 뒤 재판정한다. 판정 결과는 작업 보고에 남긴다(dogfooding ledger 입력).

## 미고정 영역 (프로젝트·상황 소유)

- 팔레트·그림자·radius·밀도·타이포 스케일: 프로젝트 DESIGN.md/토큰 소유. 그림자 허용 예외(오버레이·드래그 등)도 프로젝트가 정한다.
- 데이터 밀집 표면(대시보드·운영 도구)의 밀도: 고정 규칙 없음 — 표면 목적이 정한다.
