# Figma Bridge Candidate Horizon

Date: 2026-07-05
Status: candidate, not active

## Why This Exists

Figma is both a competitor and a complement for Askewly Design.

It competes because designers already use Figma as the default visual source of truth. It complements Askewly Design because Figma can remain a high-fidelity authoring and inspection surface while this repo focuses on reusable product UI patterns, DESIGN.md rules, implementation recipes, and coding-agent verification.

## Candidate Goal

Define how Askewly Design should connect with Figma without becoming a Figma clone.

The expected output is a bridge between:

- Figma visual artifacts and components
- Askewly Design tokens, patterns, and page recipes
- Codex / Claude Code implementation workflows
- Chrome verification evidence from the rendered app

## Candidate Questions

- Which Figma assets should become Askewly Design references, and which should remain external?
- Can DESIGN.md be generated from selected Figma styles, components, or frames?
- Can Askewly Design examples be exported back into Figma as reviewable artifacts?
- Where does Figma remain the stronger tool, and where should Askewly Design deliberately differ?

## Non-Goals

- Do not make Askewly Design a generic Figma file gallery.
- Do not treat Figma as the only design source of truth.
- Do not prioritize this before PSS2 landing quality is stable.
