# Horizon - Design System Foundation

## Why Now

The project started as a UI vocabulary encyclopedia and has already accumulated Tailwind/Tailwind Plus parity work, documentation pages, interaction examples, screenshots, and research ledgers. The product direction has now expanded: this should become Yusung's broader digital product design system, not just a web UI dictionary.

This horizon converts that direction into durable project contracts before more reference capture or implementation work continues.

## Goal

Align the repository around a digital product design system that serves both humans and coding agents.

## Milestone Plan

- DSF1 - Objective and Product Contract: update Objective, PRD, Architecture, ROADMAP, and ADR.
- DSF2 - Reference Strategy: define how Tailwind, Tailwind Labs repositories, product sites, mobile guidelines, SaaS examples, and local design-manual/custom-skills work are studied and absorbed.
- DSF3 - Surface Taxonomy: define the top-level surfaces and navigation model beyond Tailwind Plus categories.
- DSF4 - Agent-Usable Design System Model: define how examples become tokens, recipes, prompts, code assets, and downloadable artifacts.

## Closing Criteria

- ROADMAP has an active harness goal and milestone markers.
- The product documents no longer describe the project as only a UI vocabulary encyclopedia.
- Future Tailwind or design-manual research has a clear place in the system.
- At least one follow-up milestone can be executed without redefining the project again.

## Objective Impact

This horizon moves the project from vocabulary catalog toward design-system foundation by making the broader cross-surface, human-facing, and agent-facing goal explicit.
