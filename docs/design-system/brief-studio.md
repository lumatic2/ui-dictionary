# Brief Studio — 실물 선택 브리프

Date: 2026-07-19
지위: 정본. design-brief.md(착수 인터뷰 계약)의 시각 선택 경로가 이 문서를 가리킨다. AskewlyDesign 데스크톱 앱 브리프 화면의 웹 프로토타입.

디자인 결정을 텍스트 라벨("웜 페이퍼 + 그린")로 내리게 하지 마라 — 컬러·타이포·인터랙션은 **눈으로 보고 만져보고 고르는 것**이다. 풀 브리프에서 이 세 도메인은 스튜디오 선택이 기본 경로, 텍스트 질문은 폴백이다.

## 1. 에이전트 구동 절차

1. 템플릿 2파일을 작업 폴더로 복사: `templates/brief-studio.html` + `templates/brief-studio-server.py` (askewly 레포 로컬이 없으면 텍스트 인터뷰 폴백).
2. **후보 커스터마이즈**: HTML의 `AGENT:` 주석 지점에서 후보를 브리프 맥락에 맞게 교체한다 — 컬러 팔레트 3~4개(각각 미니 UI 프리뷰에 실값 적용), 폰트 3~4종(Google Fonts CSS API, 스펙시멘 문장은 프로젝트 문구로), 인터랙션 3수위(살아있는 데모). 그룹당 추천 1개(`data-rec`)를 톤 답변 근거로 지정.
3. `python brief-studio-server.py <port>` 구동 → **사용자 기본 브라우저로 열기** (Windows `Start-Process`, mac `open`).
4. `brief-selections.json` 생성을 폴링(백그라운드) — 사용자가 "선택 완료"를 누르면 생성된다.
5. 선택값을 DESIGN.md(공식 Stitch 스키마)로 매핑: 팔레트 → `colors:` 실값, 폰트 → `typography.*.fontFamily`, 인터랙션 수위 → Do's and Don'ts·모션 프로즈 + expressive-stack 티어 상한.
6. 확인 후 서버 종료·임시 파일 정리.

## 2. 폴백 (스튜디오 실패 ≠ 브리프 중단)

- **서버 미가동/전송 실패**: 페이지가 선택 JSON을 textarea로 표시 — 사용자가 채팅에 붙여넣으면 동일하게 처리한다.
- **오프라인(폰트 fetch 실패)**: 스펙시멘이 시스템 폰트로 렌더 — 페이지는 깨지지 않으며, 서체 도메인만 텍스트 질문으로 전환한다.
- **브라우저를 못 여는 환경(headless)**: design-brief.md §4 폴백(추천 기본값 + 추정 명시) 적용.

## 3. 후보 큐레이션 규칙

- 후보는 무작위가 아니라 **정체성·톤 답변(1도메인)의 파생**이다 — 톤이 "따뜻함"이면 팔레트 후보에 순수 다크를 넣지 말고 온도 변주를 보여라.
- 팔레트 후보는 반드시 같은 미니 UI(제목·보조 텍스트·버튼·카드)에 입혀 비교 가능하게 — 색상환·스와치 나열 금지 (coolors 링크는 사용자가 직접 탐색을 원할 때 보조로만).
- 인터랙션 "풍부" 선택 시에도 실험적 표현(③·④ 티어)은 별도 수동 승인 원칙 유지 (expressive-stack 판정 절차 5).
- 딥 브리프 발동 시(사용자 요청·화면 3개 이상) 컴포넌트·IA 그룹을 같은 패턴으로 추가한다 — design-brief.md 부록 참조.
