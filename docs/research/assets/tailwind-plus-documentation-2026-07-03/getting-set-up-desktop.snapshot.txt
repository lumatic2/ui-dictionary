    - heading "Requirements" [level=2]
    - paragraph: "All of the components in Tailwind Plus are designed for the latest version of Tailwind CSS, which is currently Tailwind CSS v4.2. To make sure that you are on the latest version of Tailwind, update via npm:"
    - code: npm install tailwindcss@latest
    - paragraph:
      - text: If you're new to Tailwind CSS, you'll want to
      - link "read the Tailwind CSS documentation":
        - /url: https://tailwindcss.com/docs
      - text: as well to get the most out of Tailwind Plus.
    - heading "Add the Inter font family" [level=2]
    - paragraph:
      - text: We've used
      - link "Inter":
        - /url: https://rsms.me/inter
      - text: for all of the Tailwind Plus examples because it's a beautiful font for UI design and is completely open-source and free. Using a custom font is nice because it allows us to make the components look the same on all browsers and operating systems.
    - paragraph: "You can use any font you want in your own project of course, but if you'd like to use Inter, the easiest way is to first add it via the CDN:"
    - code:
      - generic: <link
      - text: rel
      - generic: ="stylesheet"
      - text: href
      - generic: ="https://rsms.me/inter/inter.css"
      - text: />
    - paragraph: "Then add \"InterVariable\" to your \"sans\" font family in your Tailwind theme:"
    - code:
      - generic: "@theme"
      - text: "{ --font-sans: InterVariable, sans-serif; --font-sans--font-feature-settings: 'cv02', 'cv03', 'cv04', 'cv11'; }"
    - paragraph:
      - text: If you're still on Tailwind CSS v3.x, you can do this in your
      - code: "`tailwind.config.js`"
      - text: "file:"
    - code: "const defaultTheme = require('tailwindcss/defaultTheme') module.exports = { theme: { extend: { fontFamily: { sans: ['InterVariable', ...defaultTheme.fontFamily.sans], }, }, }, // ... }"
    - heading "Dark mode support" [level=2]
    - paragraph:
      - text: If you're using dark mode components, add the
      - code: "`dark:scheme-dark`"
      - text: class to your
      - code: "`<html>`"
      - text: element to ensure that the browser renders scrollbars and other native UIs correctly in dark mode. Also include the
      - code: "`dark:bg-gray-950`"
      - text: "class to provide a dark background for the entire page:"
    - code:
      - generic: <html
      - text: class
      - generic: ="bg-white dark:bg-gray-950 scheme-light dark:scheme-dark"
      - text: ">"
