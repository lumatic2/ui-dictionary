# 최종 보고서 — DOG6 마무리 절차가 셋을 다 부른다

> 2026-07-22 · horizon `design-output-gates` · milestone DOG6 · changesets 264·265·266

## 1. 문제 정의

DOG1이 검사기 오탐을 없앴고, DOG2가 npm에 올렸고, DOG3이 타이포 규칙을 더했고, DOG4·DOG5가 인쇄·슬라이드 근거를 에이전트 경로에 실었다.

**그런데 아무도 그것들을 부르지 않았다.**

착수 실사가 확인한 두 결함:

- **결함 5** — `askewly-design` 스킬의 마무리 절차에 검사기 호출이 **없다.** 직전 horizon VL8과 같은 병리: 자산은 있는데 에이전트 경로 밖.
- **결함 7** — 스킬 description이 ① 삭제된 전역 CLAUDE.md 절을 가리키고 ② 인쇄·PDF·슬라이드를 "이 스킬의 대상이 아니다"라고 배제한다. Objective 확장으로 그 매체들이 범위에 들어왔으므로 둘 다 거짓이 됐다.

이 horizon에서 **가장 중요한 milestone**이다. 검사기가 아무리 좋아져도 안 불리면 그대로다.

## 2. Objective 연결

"에이전트가 직접 쓰는 구현 시스템". 부르지 않는 자산은 참고 자료이지 시스템이 아니다.

## 3. 경로

3 step: 매체 분기 → 호출 배선 + 스킬 정합 → **실제로 한 바퀴 돌리기**.

## 4. 구현 결과

**매체 판정을 절차의 첫 단계로 넣었다**(`N-0. Pin the medium`). 질문 하나 — "이 산출물은 상호작용하는가?" 마무리 게이트(step 4)는 화면/인쇄/발표로 갈렸고 세 줄이 서로의 기준을 베끼지 않는다.

맨 앞에 둔 이유: **매체는 무엇을 검사할지가 아니라 무엇을 만들지를 바꾼다.** 종이에는 hover도 다크모드도 없고 슬라이드에는 반응형 브레이크포인트가 없다. 끝에서 정하면 다시 만들어야 한다.

**검사기 호출은 `4.5`로 들어갔다** — 사람 확인 게이트(5) **직전**이다. 핵심은 호출이 아니라 그 다음 문장이다:

> 위반이 나오면 그 목록을 받아 **고치고 다시 실행한다.** 보고에 첨부하고 넘어가지 않는다 — 이 단계의 목적은 사람에게 보고할 거리를 만드는 게 아니라 그 전에 스스로 고치는 것이다.

Kraft 원문이 말하는 것도 이것이다. 채점기는 CI도 사람 리포트도 아닌 **에이전트 파이프라인의 Verify 단계**에서 돈다. "보고 첨부"로 끝내면 검사기는 결함 목록 생성기가 된다.

**스킬 description을 고쳤다** — "대상이 아니다"를 **"대상이다, 다만 게이트가 다르다"**로. 규칙 본문은 복제하지 않고 entry-protocol N-0과 medium-taxonomy를 가리킨다.

**그리고 실제로 돌렸다.** 이것이 step-3이고 이 milestone의 실표면이다.

| 단계 | 결과 |
|---|---|
| 1차 | **5건**, exit 1 |
| 수정 | 색 리터럴 4건 → semantic 토큰 · 글자 크기 7단계 → 5단계 |
| 2차 | **0건**, exit 0 |

실행 경로는 npm 공개 배포본 `npx @askewly/design@0.2.0` — 레포 내부 빌드가 아니라 남의 프로젝트가 받는 그것이다.

## 5. 이슈와 해결

### DOG1의 "사소해 보이던" 수정이 여기서 값을 했다

1차 실행의 `PricingCard.tsx:18`에서 `raw-color-fn`과 `typography-scale-exceeded`가 **각각** 보고됐다. DOG1 step-3이 삼항 분기를 규칙 배열로 바꾸지 않았다면 하나만 나왔을 자리다.

그때 이렇게 적었다 — *"exit code 는 어차피 1이라 실패 여부는 안 바뀌었지만 보고가 불완전했다."* **자가 수정 루프에서는 보고 누락이 곧 수정 누락**이다. 에이전트가 목록대로 고치고 "다 고쳤다"고 판단한다.

### 미커밋 파일을 git으로 되돌리려 했다 — 두 번째

probe를 위해 수정 전 fixture로 돌아가려고 `git show HEAD:<path>`를 썼다. 그 파일은 한 번도 커밋된 적이 없어 결과가 비었고, 리다이렉트가 **파일을 0바이트로 만들었다.**

DOG5 step-2의 `git checkout` 사고와 같은 부류다. **git은 커밋된 것만 되돌린다.** 미커밋 원복은 명시적 사본이나 역치환으로 한다.

### 치환이 조용히 no-op 이었다

evidence에 배포본 확인을 덧붙이는 python 치환 하나가 대상 문자열 불일치로 아무것도 바꾸지 않았고, assert가 없어 성공한 것처럼 보였다. DOG1 step-2에서 probe 자체가 no-op이었던 것과 같은 부류다. 다시 넣을 때는 assert를 걸었다.

## 7. 후속 제안

- **인쇄·발표 게이트는 아직 돌지 않았다.** 실증한 것은 화면 매체의 루프뿐이다. 래스터화 확인·프리셋 대조는 절차에 적혔을 뿐 — 슬라이드 청사진과 인쇄 산출물이 각각 0건이라 먹일 입력이 없다. **"적혀 있다"와 "돌아간다"의 간극이 두 매체에 여전히 남아 있다.**
- **fixture는 파일 하나짜리 인공 예제다.** 실제 프로젝트에서 성가신지, 오탐이 몇 건인지는 이것이 답하지 못한다 — DOG7의 몫.
- **경고 → 차단 승격**은 DOG7 관측 뒤 사용자 판정.
- **규칙이 바뀌면 재배포하고 이 실증을 다시 돌려야 한다.** `0.2.0` 고정 실행이다.
