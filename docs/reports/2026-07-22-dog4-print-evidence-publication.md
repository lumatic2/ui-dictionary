# 최종 보고서 — DOG4 인쇄 근거가 에이전트 경로에 실린다

> 2026-07-22 · horizon `design-output-gates` · milestone DOG4 · changesets 258·259·260

## 1. 문제 정의

인쇄 규격 계약은 **이미 있었다** — `packages/template-core/src/print-spec.ts`에 명함 3규격, ISO A계열 4규격, 도련·안전영역·게시 여백·300dpi 환산, 그리고 검증 API까지.

그런데 에이전트가 fetch하는 경로(`docs/design-system/` → `llms.txt`) **밖에** 있었다. 갭의 정체는 "지식이 없다"가 아니라 **"지식이 에이전트 경로 밖에 있다"**이다. 직전 horizon VL2와 같은 구조 — 용어 562개가 사전 파일에는 있었지만 인덱스 밖이라 에이전트에게는 존재하지 않았다.

착수 실사가 후보 본문의 "지면 근거 0건"을 **절반만 맞다**고 정정한 것이 이 milestone의 출발점이었다.

## 2. Objective 연결

Objective가 2026-07-21에 범위를 넓혔다 — 화면 UI에 더해 슬라이드·지면 산출물까지. 자산이 따라오지 않으면 넓힌 범위는 선언으로만 남는다.

## 3. 경로

3 step: 계약을 문서로 승격 → 매체 분류 축 신설 → 등재와 배포본 검증.

## 4. 구현 결과

**생성물로 만들었다.** `docs/design-system/print-spec.md`는 손으로 옮긴 사본이 아니라 `scripts/generate-print-spec-doc.mjs`가 컴파일된 `template-core/dist`를 import해 값을 뽑은 결과다. 표의 300dpi 픽셀 열도 문서가 계산하지 않고 정본 함수 `mmToPrintPx()`를 부른다.

**사본이면 코드가 바뀐 뒤에도 문서는 옛 숫자를 말한다. 그리고 에이전트가 읽는 것은 문서다.** `--check`가 재생성 결과와 커밋본을 대조한다.

**근거의 한계를 값과 함께 옮겼다.** A계열 안전영역 3mm는 유추이고(포스터 전용 수치의 1차 출처를 못 찾았다), 재단 표시 선 길이는 출처가 없다. 근거 없는 수치를 근거 있는 것처럼 실으면 그걸 읽은 에이전트가 인쇄 발주를 낸다.

**매체 분류 축** — 화면/인쇄/발표 3매체 × 대표 산출물·정본 문서·게이트. 계기는 2026-07-20 어록 PDF: 화면용 warm canvas 토큰을 지면에 깔아 `@page` 여백이 흰색으로 잘렸는데 **화면 게이트는 전부 통과했다.** 게이트가 틀린 게 아니라 틀린 자를 댔다.

핵심은 표가 아니라 세 줄이 다른 이유다 — 화면은 상태를 놓쳐 깨지고, 인쇄는 물리로 깨지며 되돌릴 수 없다(잘못 발주한 1,000장), 발표는 거리·조명으로 깨진다(가설). 인쇄 게이트에 두 줄을 박았다: **지면 기본은 흰 종이다**, **최종 형식 그대로 래스터화해 본다**(HTML 스크린샷은 PDF가 아니다).

**발표 행은 비워두되 비어 있다고 적었다.** 빈칸이면 읽는 쪽이 누락인지 미정인지 모르고, 그럴듯하게 채우면 근거 없는 게이트가 근거 있는 것처럼 돈다.

**등재** — `llms.txt`에 `Media` 섹션 신설, 분류를 먼저 놓았다. 어느 자를 대야 하는지 모르면 인쇄 규격을 읽어도 화면 체크리스트를 그대로 돌린다.

## 5. 이슈와 해결

### plan의 Verify 커맨드가 이 레포에서 동작하지 않았다

`npm run build --workspace packages/template-core` — 루트 `package.json`이 workspaces를 선언하지 않는다(의도적: "하위 packages는 각자 package.json을 유지한다"). `npm --prefix` 로 대체했다. 계획이 실행 환경을 확인하지 않고 관용 커맨드를 적은 경우다.

### HTTP 200 에서 멈출 뻔했다

step-3 Verify는 200 확인까지만 요구한다. 그런데 **200은 문서가 도달했다는 증거지 내용이 온전하다는 증거가 아니다.** DOG2에서 정확히 같은 종류의 착각을 했다 — 패키지 존재(HTTP 200)를 버전 존재로 읽어 거짓 통과가 났다. 그래서 배포본에서 실제 값(85×55 행의 `1004×650px`, 환산 상수 `11.811`)이 읽히는지까지 확인했다.

### Failure probe 둘 다 실패를 봤다

이 horizon에서 probe가 두 번 자기 무용함을 드러냈던 터라 둘 다 실제 실패 출력을 확인했다: 드리프트 probe는 `exit 1` 후 원복 `exit 0`으로 **왕복** 증명(실패만 보면 항상 실패하는 검사기와 구분되지 않는다), 게이트 probe는 4건 실패.

## 6. 결과물과 증거

**크기 회고**: changeset **3개**(258·259·260)로 닫혔다. 선언과 일치 — 인플레·디플레 없음.

- 실표면: **공개 배포본**이 실표면이다. `https://ui.askewly.com/llms.txt`에 두 링크 등장(42·43행), 두 문서 HTTP 200, 배포본 본문에서 실제 규격 값 확인. Cloudflare 전파는 15초 폴링 15회(약 3분 30초).
- 재현: `npm --prefix packages/template-core run build` → `node scripts/generate-print-spec-doc.mjs --check`(exit 0) → `node scripts/generate-llms-txt.mjs`(exit 0, 159 assets) → `npm run verify`(exit 0).

증거: `evidence/design-output-gates/dog4-print-publication.md`

## 7. 후속 제안

- **등재 ≠ 호출.** 이 milestone은 근거를 경로에 실었을 뿐이고, 마무리 절차가 매체에 따라 이 문서를 지시하게 하는 것은 **DOG6**이다. 지금 상태로는 에이전트가 스스로 찾아 읽어야 한다.
- **래스터화 게이트는 기계 검증이 없다.** 색·타이포와 달리 `verify`가 잡지 못하는 절차 항목이다. 자동화 가치는 DOG7 사람 관측 입력으로 남긴다.
- **발표 칸은 DOG5가 채운다.** 그때까지 슬라이드 작업의 화면 게이트 준용은 품질을 보증하지 않는다.
