# 최종 보고서 — ECT4 이미지 노드가 실제로 칠해진다

> 완료: 2026-07-21 · 대상: ECT4 · 작성: 완료 경계(§B3)
> 배치: `archive/reports/2026-07-21-ect4-image-node-and-contract.md` (record — 작성 후 동결)

## 1. 문제 정의

두 가지였다. 하나는 이미지 노드가 색 바인딩을 렌더에 반영하지 않는다는 것,
다른 하나는 허용 바인딩 키의 목록이 여러 곳에 흩어져 한쪽만 고치면 나머지가 뒤처진다는 것.

**첫 번째는 사실이 아니었다.** 그 발견이 이 milestone의 실질이다.

## 2. Objective 연결

"사람이 코드 네이티브 캔버스에서 실제 컴포넌트를 조작한다"는 축의 마무리 층 —
사람이 만든 색 결정이 화면까지 손실 없이 도달하는지, 그리고 그 규칙이 한 곳에서만 정의되는지.

## 3. 경로

step-1(키 정본 통일)과 step-2(이미지 노드)를 밟았고, 사용자 지시로 이월 finding 마감이
붙어 3 changeset이 됐다.

**step-2는 착수 직전에 성격이 바뀌었다.** 전제를 실측했더니 거짓이어서, "렌더러 확장"이
아니라 "틀린 전제로 막아둔 것을 열고 렌더 테스트로 고정"하는 일이 됐다.

## 4. 구현 결과

허용 바인딩 키의 정본이 `canvas-core` 하나가 됐다. `template-core`·렌더러·인스펙터가 모두
그것을 읽고, 어긋나면 계약 테스트가 실패하거나 인스펙터가 개발 중 즉시 예외를 던진다.

이미지 노드도 색을 묶을 수 있다 — 원래 칠해지고 있었는데 잘못된 실사로 막아뒀던 것을 열었다.

인스펙터가 한 언어로 말한다. 표시는 한국어, 값은 문서 모델 그대로다.

## 5. 이슈와 해결

**전제가 거짓이었다. 그리고 그 거짓이 사용자 결정까지 갔다.**

세션 첫 실사(코드 탐색)가 "렌더러가 image kind에서 `tokenBindings`를 아예 참조하지 않는다"고
적었다. 실측하니 `<img>`도 `shared.style`을 받아 배경 바인딩이 **처음부터 칠해지고 있었다** —
인라인 배경이 토큰 해석값과 일치한다.

그 한 문장이 네 곳으로 전파됐다: horizon 문서의 4표면 논거, ECT3의 이미지 노드 제외,
ECT4 step-2의 존재 이유, 그리고 **사용자 결정 3**. 사용자가 거짓 전제 위에서 선택했다.

EU5에서 이미지 노드 색을 못 바꾼 진짜 이유는 렌더러가 아니라 **바인딩이 없고 묶을 경로도
없었던 것**이며, ECT3의 묶기 어포던스가 이미 해결한 문제였다. 결정 3이 요구한 결과는
이미 충족돼 있었다.

**왜 이제야 잡혔나** — 첫 실사는 코드를 *읽어* 판단했고, 실제 값은 공통 `shared` 객체를 통해
흘러들어오고 있었다. 읽어서는 놓치기 쉬운 형태다. 렌더해서 재니 5분이었다.
horizon 문서와 plan에 정정을 붙였다(plan 본문은 승인 hash 보존을 위해 상단 정정 블록으로).

**측정 방법이 판단을 뒤집은 게 이 horizon에서 세 번째다** — 어휘 이름 겹침(ECT1),
WebGPU 색(ECT2), 이미지 렌더(ECT4). 세 번 다 "읽어서 그렇다고 생각한 것"이 "재보니 아니었다"였다.

**중간에 내 커밋이 검증자의 임시 파일을 쓸어담았다.** `git add -A`가 같은 작업 트리에서 도는
검증 세션의 파일까지 담았고, 검증자가 알려줘서 별도 커밋으로 제거했다.

## 7. 후속 제안

- **ECT5는 사용자 브라우저가 필요하다** — 2026-07-21 현재 사용자가 브라우저를 쓸 수 없어
  **여기서 정지·핸드오프**한다. 스크린샷 제시로 대체하지 않는다(직전 horizon에서 실패한 방식).
- **실사 단계를 "읽기"에서 "실행해서 재기"로 바꾼다** — 이 horizon에서 세 번 뒤집혔다.
  다음 horizon 설계의 실사에 이 규약을 넣는다.
- **어휘 등록의 단일 방어선**(ECT1 이월) — `registerTokenVocabulary` 미등록 시 조용히 강등되고
  배선 테스트 하나가 유일한 방어선이다. 구조적 강제는 결정 사안으로 남는다.
- **`git add -A` 습관** — 같은 작업 트리에서 다른 세션이 돌면 남의 파일을 담는다.
  경로를 명시하거나 커밋 전 `git status`를 읽는다.
