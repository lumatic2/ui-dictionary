## Goal

스튜디오를 **폭포식 결정 흐름**으로 재구축한다: 영향 큰 축부터 위→아래로 배치하고, 위 선택이 아래 후보 카드에 **즉시 입혀지며**(배경을 고르면 액센트 후보가 그 배경 위에서 렌더), 스크롤을 따라오는 **스티키 라이브 미리보기 패널**에서 미니 사이트가 선택마다 점진적으로 완성된다. 결함 정비(칩 겹침·카드 크기·사진 해상도), 영상 후보(Pexels Videos), 카피·인터랙션 축 추가 포함.

## Why Now

- 사용자 피드백 2026-07-19 (댄스 뮤지컬 실연 직후, 6건+1): ① 순위 칩이 콘텐츠와 겹침 ② 타일·이미지 카드 더 크게 ③ 영향 순 배치 + 위 선택이 아래에 바로 적용 ④ 사진 해상도 낮음 ⑤ 영상이었으면 ⑥ 히어로 문구·설명·CTA 개수·마우스 인터랙션도 선택 가능하게 ⑦ (승인 시 추가) 스크롤 따라오는 미리보기로 폭포가 완성되는 사이트를 보며 진행.
- studio-depth SP4 갭 3건과 합류(후보 주입·이미지 규칙·폴백 규칙).

## 설계 원칙

- **폭포 순서 = 영향 크기**: 방향 타일 → 배경 → 액센트 → 타이포 → 이미지/영상 → 구성 → 헤더/푸터 → 카드 → 밀도/radius → 카피(헤드라인·서브·CTA) → 인터랙션 → 다크/접근성.
- **캐스케이드 렌더**: 각 그룹 후보는 "지금까지의 선택"을 컨텍스트로 렌더 — 아직 안 고른 축은 추천 1순위를 임시값으로.
- **스티키 미리보기**: 우측(모바일은 하단) 고정 패널에 미니 사이트(헤더·히어로·카드·푸터 축약판)가 선택 상태로 실시간 조립.
- 해상도 계약: 스튜디오 썸네일=medium, 최종 페이지=large2x/원본. 영상은 HD + 포스터 폴백(reduced-motion·모바일).
