# Horizon - Reference Model Execution

## Why Now

The previous horizon defined the product direction, reference strategy, surface taxonomy, and agent asset model. The next gap is execution: references must be converted into reusable design-system models rather than remaining as screenshots, parity notes, or loose inspiration.

Tailwind/Tailwind Plus should be consolidated first because the repo already contains substantial capture and implementation evidence. Around/Createx should be added immediately after because it represents a different kind of reference: a multipurpose commercial template system with broad landing/page/account/UI Kit/docs coverage and theme customization controls.

## Goal

Create a repeatable reference-to-design-system pipeline:

reference source -> evidence capture -> transferable principles -> non-transferable identity -> surface/pattern/component mapping -> implementation target -> agent recipe implications.

## Milestone Plan

- RME1 - Tailwind Reference Model: consolidate existing Tailwind/Tailwind Plus evidence into a reusable model.
- RME2 - Around Template System Capture: capture Around as a multipurpose template-system reference and compare it against the surface taxonomy.
- RME3 - Local Design Work Audit: inspect local design-manual/custom-skills work and decide what to merge, link, supersede, or archive.
- RME4 - Mobile Platform Baseline: capture HIG/Material mobile expectations to prevent the system from staying web-only.
- RME5 - Product-System Exemplar Map: map Vercel, Stripe, Linear, Radix/Linear, and Around into product-system lessons.

## Decision Log

- Around/Createx is now an explicit reference target.
- The Tailwind Labs repo deep dive remains useful, but it is not the first milestone in this horizon. Site/docs/Plus evidence is more immediately actionable.
- Each reference output must identify what can be adapted and what must not be copied.
- Implementation is out of scope until at least RME1 and RME2 produce reference models that can feed the surface taxonomy and agent asset model.

## Closing Criteria

- At least Tailwind and Around have dedicated reference model documents.
- Local design work is classified enough to avoid duplicate design-system effort.
- Mobile baseline exists before mobile UI implementation begins.
- Product-system exemplars are mapped into the surface taxonomy and agent asset model.

## Objective Impact

This horizon moves the Objective from strategic definition into evidence-backed reference intake. It should make future implementation less ad hoc by turning external and local references into structured models that humans can browse and agents can use.
