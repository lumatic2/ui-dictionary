## Amendments

1. One YAML flow-mapping pitfall hit during Adapt, the same failure mode `documentation-depth`'s Amendment 1 first documented: an unquoted flow-mapping `sources` entry's `note:` value for the new `onboarding-carousel-skip-pattern` term contained an embedded comma ("onboarding should be brief and skippable, and should not force users to repeat it"). This parsed cleanly through `validate-ui-vocabulary.py`'s Python YAML loader but then failed at `tsc` on the generated `terms.generated.ts` with "Object literal may only specify known properties" — the comma was read as introducing a second flow-mapping key. Fixed by rephrasing the note to drop the embedded comma, consistent with the standing "no embedded punctuation inside unquoted flow scalars" guidance. Confirms this failure mode recurs across batches and is worth promoting from a per-batch amendment into `reference-loop.md`/`recipe-format.md` guidance directly in a future procedural pass.
2. Staging the inbox and running the dedup audit strictly before authoring any recipe/term files (per the corrected stage order) produced a clean pre-adapt audit (exit 0, no self-match errors) for the second consecutive batch — reconfirms `documentation-depth`'s and `internal-tools-depth`'s own recommendation.
3. This batch is the first SD-depth batch with 4 judgment candidates requiring a genuine manual read rather than a fully pre-adjudicated list; 3 of 4 confirmed the orchestrator's stated default (enrichment), and 1 (`onboarding-carousel-skip-pattern`) diverged from the default because the neighbor term's anatomy had no coverage at all of the candidate's core behavioral contract (skip persistence) — worth noting for future batches that a near-name/alias collision (flagged by the audit or a manual alias read) does not by itself mean the underlying behavior is already covered; the two checks (name/alias overlap vs. behavioral/anatomy overlap) can point in different directions.
