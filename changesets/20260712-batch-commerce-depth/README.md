## Amendments

1. A `description` field for `cross-sell-related-strip` originally embedded double-quoted Korean phrases inside an unquoted plain (non-flow) YAML scalar (e.g. `Ìï®Íªò Íµ¨Îß§ÌïòÎ©¥ Ï¢ãÏùÄ", "Í¥ÄÎÝ® ÏÉÅÌíà Í∞ôÏùÄ ‚Ä¶`) and broke `validate-ui-vocabulary.py`'s YAML parser with a "while parsing a block mapping" error. This is the same category of pitfall as the prior batches' documented flow-sequence quote issue, but manifesting in a plain scalar this time ‚Äî resolved by rephrasing to drop the embedded quote marks entirely rather than selectively escaping them. Worth folding into `reference-loop.md`'s existing YAML-quoting note: avoid embedded double quotes inside *any* unquoted YAML scalar (plain or flow), not just flow-sequence items.
2. This batch's required manual neighboring-`terms.yml` read (authoring-workflow.md ¬ß3, applied via RL) surfaced two near-duplicates the collector's own `dedup_hints` did not flag: `delivery-tracker`'s overlap with the standalone `timeline` term, and `faceted-filter`'s overlap with `filter-panel`'s pre-existing "Filter sidebar" alias. Both were caught only because every candidate's likely neighbors were read in full rather than relying on the automated audit's token/substring heuristics ‚Äî reinforcing the existing reference-loop.md guidance that dedup_hints and the automated audit are a starting point, not a substitute for reading actual neighboring entries.
3. Recipe files for the 3 fixed-adjudicated recipe promotions were authored before staging the batch into `docs/research/loop/inbox.yml` and running the pre-adapt dedup audit (rather than after, per the normal stage order). This caused the audit run in this batch to report 3 expected self-match errors instead of a clean exit 0. The dedup substance was unaffected (these 3 promotions were fixed by the orchestrator prior to this session and did not require dedup discretion), but future batches should stage the inbox and run the audit strictly before writing any recipe/component files, even when promotions are pre-adjudicated, to keep the pre-adapt audit output clean.
