## 설계 판단 — 검색은 필터가 아니라 트리 유지다

매칭된 행만 남기면 계층이 사라져 **"어디 있는 건지"를 못 답한다.** 그건 검색이 아니라 목록이다.
그래서 매칭 행 + **그 조상 전부**를 함께 보이고, 조상은 강제로 펼친다(접힌 조상 아래 결과가
숨으면 찾은 게 아니다).

- 매칭 행은 굵게, 조상은 흐리게 — `data-layer-match="hit" | "context"`로 구분한다.
- 결과 건수를 숫자로 보인다. 0건이면 빈 트리로 넘기지 않고 없다고 말한다.
- 검색은 패널 로컬 상태다 — 문서를 바꾸지 않고 연산도 만들지 않는다.
