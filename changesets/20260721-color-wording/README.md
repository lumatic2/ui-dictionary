# 색 UI 문구 — 내부 설계 언어를 화면에서 걷어낸다

> 2026-07-21 · milestone ECT5 step-2 · changeset #245

## 왜

ECT5 사람 관측에서 사용자가 막힌 곳이 기능이 아니라 **어휘**였다:

> *"색을 묶으라는게 무슨 말이지? 색을 붙인다는 건 뭐고?"*

그 어휘는 에이전트가 지어낸 게 아니라 **화면에 박혀 있던 문구**다. ECT3 milestone 제목
("묶고 푼다")에서 코드로, 코드에서 화면으로 검문 없이 흘러내렸다.
기능 3건은 픽셀 실측으로 정상 확인됐다(`ect5-judgeability.md`) — **못 넘은 것은 말이다.**

## 사용자 결정 (2026-07-21)

1. **축 = "토큰 유지, 동사만 교체".** `토큰`은 화면에 남기고 `묶다/풀다/벗어나다`만 바꾼다.
   제품 전체(DESIGN.md·에이전트 문서)와 어휘가 하나로 유지된다.
2. **범위 = 인접 문구까지.** 관측에서 실제로 막힌 4건뿐 아니라 같은 결함(내부 용어 노출)인
   해석 실패 안내문·입력칸 라벨까지 통일한다.

## 무엇을

`apps/agent-design/src/PropertyInspector.tsx` 문구 7건:

| 위치 | 전 | 후 |
|---|---|---|
| detach 버튼 | `토큰에서 풀기` | `직접 색 지정` |
| bind 버튼 | `{label} 묶기` | `색 고르기` (aria `{label} 고르기` 신설) |
| 직접지정 안내 | `토큰에서 벗어난 색이다. 견본을 눌러 다시 묶을 수 있다.` | `토큰을 쓰지 않고 직접 지정한 색입니다. 견본을 누르면 토큰으로 돌아갑니다.` |
| 직접지정 aria | `— 토큰에서 벗어남, 눌러서 다시 묶기` | `— 직접 지정한 색, 눌러서 토큰으로 되돌리기` |
| 미해결 안내 | `이 토큰은 지금 문서의 토큰 세트에서 해석되지 않는다.` | `이 이름의 토큰이 지금 문서에 없습니다.` |
| 미해결 aria | `해석되지 않음` | `없는 토큰` |
| 입력칸 aria | `{label} 원시 값` | `{label} 직접 지정 값` |

**화면 텍스트와 `aria-label`을 같은 축으로 함께 바꿨다** — 화면만 고치면 스크린리더
사용자에게만 옛 어휘가 남는다.

`COLOR_BINDING_LABELS` 위에 **어휘 분리가 의도된 것**임을 주석으로 남겼다. 코드·testid·
에이전트 문서는 계속 "binding·묶다·detach"로 말한다. 다음 사람이 불일치로 오인해
되돌리지 않게 evidence 경로를 함께 적었다.

## 바꾸지 않은 것 (판단 기록)

- **상단 `토큰 세트` 선택기 라벨** — 결정 2의 "인접 문구" 대상으로 검토했으나 그대로 뒀다.
  결정 1이 `토큰`을 화면에 남기기로 했고, 이 라벨에는 금지 동사가 없다. 바꿀 이유가 없다.
- **동작·구조** — 문자열과 aria만 건드렸다. 연산·모델·렌더 변경 0건.
- **테스트 이름의 한국어 서술**("안 묶인 것을 묶는다" 등) — 개발자가 읽는 기계 어휘라 유지.
