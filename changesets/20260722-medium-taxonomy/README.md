# 매체 분류 축 — 화면의 자로 지면을 재면 지면 고유의 실패가 통과한다

> 2026-07-22 · milestone DOG4 step-2 · changeset #259

## 왜

2026-07-20 어록 PDF 사고: 화면용 warm canvas 토큰을 지면에 그대로 깔아 `@page` 여백이 흰색으로 잘렸다. **`askewly-design` 스킬의 화면 게이트는 전부 통과했는데 결과물은 깨졌다.**

게이트가 틀린 게 아니라 **틀린 자를 댔다.** 그런데 "어떤 산출물에 어떤 자를 대는가"를 말하는 문서가 없었다 — 게이트 서술이 화면 하나를 암묵 전제로 여러 문서에 흩어져 있었다.

## 무엇을

`docs/design-system/medium-taxonomy.md` 신설. 화면/인쇄/발표 3매체 × 대표 산출물 · 정본 문서 · **게이트**.

핵심은 표가 아니라 **세 줄이 서로 다른 이유**다. 게이트가 매체마다 다른 건 편의가 아니라 실패 방식이 달라서다 — 화면은 상태를 놓쳐 깨지고, 인쇄는 물리로 깨지며(되돌릴 수 없다: 잘못 발주한 1,000장), 발표는 거리·조명으로 깨진다(가설).

두 가지를 명시했다:

- **지면 기본은 흰 종이다.** 화면 캔버스 토큰을 그대로 옮기지 않는다 — 사고의 직접 원인.
- **최종 형식 그대로 래스터화해 본다.** HTML 스크린샷은 PDF가 아니다. `@page` 여백·페이지 분할·폰트 임베딩·색 프로파일이 전부 브라우저와 인쇄 엔진 사이에서 갈린다. 첫·중간·끝 최소 3면 — 첫 장만 보면 페이지 분할 실패가 안 보인다.

## 발표 행을 비워두되 비어 있다고 적었다

발표 매체는 DOG5 대상이다. 규격도 게이트도 아직 없다. **추측으로 채우지 않았고, 대신 그 칸이 미정이라는 사실과 "화면 게이트 준용은 지면·투사 품질을 보증하지 않는다"를 적었다.**

빈칸으로 두면 읽는 쪽이 "누락"인지 "미정"인지 모른다. 그럴듯하게 채우면 근거 없는 게이트가 근거 있는 것처럼 돈다.

## 이 문서는 생성물이 아니다

step-1의 `print-spec.md`와 지위가 다르다. 인쇄 규격은 코드가 정본이라 생성했지만, 매체 분류는 코드에 대응물이 없는 **정책 문서**다 — 정본이 곧 문서 자신이다(`pattern-taxonomy.md`와 같은 지위). 그래서 손으로 쓴다.

## 검증

행 추출 후 게이트 셀을 대조한다 (재현 가능한 검사, 스크립트는 일회성):

```bash
fullrow() { grep -m1 "^| \*\*$1" docs/design-system/medium-taxonomy.md; }
gate() { fullrow "$1" | awk -F'|' '{print $(NF-1)}'; }
# 화면=다크모드·상태 / 인쇄=래스터화·PyMuPDF·도련 / 발표=DOG5
# 그리고 세 게이트 셀이 서로 달라야 한다
```

## Failure probe

인쇄 행의 게이트 셀을 화면 행과 **같은 문구로 치환** → **4건 실패**(래스터화·PyMuPDF·도련 누락 + "게이트 서술이 겹친다"). 원복 후 통과.

표만 있고 실은 같은 자를 대는 상태를 잡는 것이 이 probe의 요지다 — 매체를 나눠 적었다는 사실만으로는 게이트가 갈렸다는 증거가 안 된다.
