# 돌렸더니 재측정이 일어났다

> 2026-07-22 · milestone DOG6 step-3 · changeset #266

## 왜

step-2가 절차에 검사기 호출을 적었다. **"절차에 적혀 있다"와 "돌렸더니 실제로 일어난다"는 다른 사실이다** — 이 horizon 프리모템 1이 정확히 그 간극을 지목한다.

## 무엇을

의도적 위반 fixture 하나로 루프를 실제로 1회 통과시켰다. 실행 경로는 **npm 공개 배포본** `npx @askewly/design@0.2.0` — 레포 내부 빌드가 아니라 남의 프로젝트가 받는 그것이다.

| 단계 | 결과 |
|---|---|
| 1차 | **5건**, exit 1 (hex ×2 · raw-color-fn ×2 · typography 7단계) |
| 수정 | 색 리터럴 → semantic 토큰 유틸리티, 7단계 → 5단계 |
| 2차 | **0건**, exit 0 |

세 조건(1차>0 · 실제 수정 발생 · 2차=0)이 다 있어야 자가 수정 루프다. 2차만 0이면 "처음부터 위반이 없었다"와 구분되지 않는다.

## 한 줄에서 두 규칙이 동시에 잡혔다

`PricingCard.tsx:18`에서 `raw-color-fn`과 `typography-scale-exceeded`가 **각각** 보고됐다. DOG1 step-3이 삼항 분기를 규칙 배열로 바꾸지 않았다면 하나만 나왔을 자리다 — 그때 "exit code 는 어차피 1이라 실패 여부는 안 바뀐다"고 적었는데, **자가 수정 루프에서는 보고 누락이 곧 수정 누락**이 된다.

## Failure probe

안 고치고 그대로 재실행 → **5건, 1차와 출력 완전 동일**(diff 없음).

2차가 0이 된 것이 *수정 때문*이지 *재실행이 항상 0을 낸다*가 아님을 이것이 가른다.

## 사고 기록 — 두 번째 같은 실수

수정 전 fixture로 되돌리려고 `git show HEAD:<path>`를 썼다. 그 파일은 **한 번도 커밋된 적이 없어서** 결과가 비었고 리다이렉트가 파일을 0바이트로 만들었다.

DOG5 step-2의 `git checkout` 사고와 같은 부류다. **git은 커밋된 것만 되돌린다. 미커밋 원복은 명시적 사본이나 역치환으로 한다.**
