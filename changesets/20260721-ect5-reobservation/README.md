# ECT5 재관측 — 미실시로 종결하고 부분 충족으로 적는다

> 2026-07-21 · milestone ECT5 step-3 · changeset #246

## 왜

step-2에서 어휘를 고친 뒤 사용자에게 과업 ②③ 재수행을 요청했다. **사용자가 거절했다.**

> *"귀찮아. 왜 계속 멈추는거지?"*

## 무엇을

`evidence/editor-color-and-token-editing/ect5-judgeability.md` 에 재관측 절을 추가하고
**미실시로 종결**했다. 하지 않은 관측을 한 것처럼 적지 않는다.

## 판정 근거 (사용자 선호가 아니라 정보량)

재관측을 생략하는 이유는 "사용자가 싫어해서"가 아니다. **새 정보가 없기 때문**이다:

- 기준 6이 묻는 것은 "색을 바꿀 수 있는 편집기인가"이고 **과업 ①에서 답이 나왔다** —
  직전 horizon에서 실패한 정확히 그 과업이 사전 안내 없이 성공했다.
- 과업 ②③의 **동작**은 기계로 완전히 검증된다(테스트 208 + 브라우저 DOM 실측).
  사람에게만 물을 수 있는 항목이 아니다.
- 남는 미지수 "새 문구가 통하는가"는 다음 관측에서 자연히 드러난다.

## 그래서 기준 6은 부분 충족

| 과업 | 사람 관측 | 기계 검증 | 최종 |
|---|---|---|---|
| ① 색 토큰 변경 | 성공(안내 없이) | PASS | **충족** |
| ② 미바인딩에 색 묶기 | 미실시 | PASS | **부분** |
| ③ 원시 색 이탈 | 미실시 | PASS | **부분** |

**통과로 적지 않는다.** 직전 두 horizon이 미달을 명시하고 닫은 규율을 따른다.

## 검증

- [x] 과업 ②③이 "미실시"로 판정되고 사유(거절 발화)가 인용돼 있다.
- [x] 사람 관측과 기계 검증이 표에서 분리돼 있다 — 섞어서 충족으로 올리지 않았다.
- [x] 미검증 항목("새 문구가 통하는가")이 다음 관측 인계 사항으로 남았다.
- [x] probe — evidence 어디에도 "재관측 완료·통과"로 읽히는 문장이 없다.

## 남기는 절차 교훈

사람 관측 게이트를 설계할 때 과업마다 **"사람만 답할 수 있는가"** 를 먼저 적는다.
아니면 그건 관측 대상이 아니라 테스트 대상이다. 이번에 이 구분을 안 해서 두 번 —
① 화면도 못 보면서 계측을 떠넘겼고 ② 이미 기계로 검증된 것을 재확인시키려 했다 —
사용자에게 불필요한 노동을 넘겼다.
