# 임계값 실측 — 임계값은 틀리지 않았다, 경계가 틀렸다

> 2026-07-22 · milestone DOG3 step-3 · changeset #256

## 왜

임계값 5는 우리 토큰 스케일(5단계)에서 유도한 값이다. 그럴듯하다고 맞는 것은 아니다. **우리 자산에 실제로 돌려서** 현실적인지 재고, 그 결과로 최종 확정한다.

## 무엇을

`examples/ui-vocabulary-site/src` 99개 파일(tsx·ts·css)에 실행 관측. **코드 변경 없음** — 재는 step이다.

## 관측

```
단계수 분포:  0→25  1→35  2→21  3→7  4→2  5→1  |  6→1  7→1  8→2  11→2  15→1  16→1
```

**91/99 (91.9%)가 5 이하.** 초과 8건.

초과 8건을 뜯어보니 둘로 갈린다:

- **5/8이 400줄 초과 다중 컴포넌트 파일.** `App.tsx`는 **23,508줄**이다. 이것을 "한 화면"이라 부를 수 없다.
- **4/8이 12px 미만 값을 2개 이상** 쓴다(7·8·9·10·11px). 쇼케이스 카드 안의 **축소된 기기 목업** — 디자인 실패가 아니라 의도된 장식이다.

**400줄 이하 파일만 보면 89개 중 3개 초과 (3.4%).**

## 판정

| 물음 | 결론 |
|---|---|
| 임계값 5가 현실적인가 | **유지.** "한 화면일 법한" 크기에서 96.6% 통과 |
| 파일 = 화면이 맞는가 | **이것이 진짜 한계다** |

**임계값은 틀리지 않았다. 경계가 틀렸다.**

23,508줄 파일이 15단계를 쓰는 건 그 파일이 화면 하나가 아니어서지 타이포 규율이 무너져서가 아니다. 임계값을 올려 이 파일을 통과시키면 **진짜 화면의 남용까지 함께 통과**한다. 그래서 올리지 않는다.

## finding 큐

- **파일=화면 경계의 한계** — 다중 컴포넌트 파일 오탐. DOG7 사람 관측에서 "오탐이 성가신가"의 1순위 대상.
