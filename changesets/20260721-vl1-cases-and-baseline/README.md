## 무엇을 했나

외부 출처 라벨 케이스 32건(12군집)을 만들고, 채점기를 세우고, 현행 자산만 쥔 격리
서브에이전트로 기준선을 측정했다.

## 결과 — 기준선 84.4% (27/32)

**예상보다 훨씬 높다. 12군집 중 8군집이 이미 만점이다.**
결정 계층의 향상 여지는 5건, 그중 진짜 오판은 3건(overlay-3·single-choice-3·waiting-3)이다.
틀린 3건의 공통점: 전부 **임계값이 있는 규칙**(스크롤이 필요할 만큼 길면 / 공간이 있으면 /
단일 모듈이면). 모델이 상식으로 아는 것과 못 아는 것의 경계가 거기다.

## 정직하게 남긴 것

- **계획 미달**: 군집 15개 → 12개. 3군집은 정답 라벨의 외부 원문을 못 찾아 뺐다. 부정문에서
  긍정 라벨을 추론하지 않았고 통념을 지어내지 않았다.
- **자를 사후에 손댔다**: 표기 동의어 표를 기준선 답을 본 뒤 추가했다. 판정이 아니라 표기만
  다른 쌍만 넣었고, 방향은 기준선을 65.6%→84.4%로 **올리는** 보수적 쪽이다.
- **케이스 결함 2건을 고치지 않고 남겼다**: floating-1(요구 문장이 답을 하나로 못 좁힘),
  floating-2(우리 레시피가 Polaris 라벨과 충돌 — 우리 쪽이 더 정확할 수 있음). 답을 본 뒤
  케이스를 손보면 그때부터 자가 아니라 답안이다.
