## Amendments (for orchestrator fold into `reference-loop.md`/authoring docs)

1. **YAML flow-sequence scalars must not contain embedded quote marks.** Two authoring mistakes hit `validate-recipes.py`/`validate-ui-vocabulary.py`'s YAML parser during Adapt: a `visual_anatomy` list item written as `["Danger Zone" heading, ...]` and an alias written as `["Login as" banner, ...]` both broke flow-sequence parsing ("while parsing a flow sequence... expected ',' or ']'"). Neither is a validator bug — an unquoted flow-list item containing an embedded double-quoted phrase is invalid YAML flow-scalar syntax. Fixed by rephrasing (`danger zone heading label`, `Login-as banner`) rather than selectively quoting. Worth a one-line caution in `authoring-workflow.md`/`recipe-format.md` so future batches don't lose time to a cryptic parser traceback.
2. **Pre-adapt dedup re-verification required a temporary stash.** Because this batch's Adapt stage (writing recipes/terms) happened before the ledger/changeset write-up, getting an accurate "pre-adapt" audit result for the record required `git stash`-ing the in-progress `terms.yml` edits and moving the new recipe files aside, running the audit against the pristine tree, then restoring. This produced the exact same exit-0/10-warning result reported at collection time, confirming the collector's pre-check was accurate. Future batches should run and capture the pre-adapt audit output *before* starting Adapt edits, to avoid this extra stash/restore step.
