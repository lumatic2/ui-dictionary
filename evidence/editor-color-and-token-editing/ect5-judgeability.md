# EVIDENCE — ECT5 판단 가능성 재관측

- Milestone: ECT5 (`plans/2026-07-21-ect5-judgeability-observation.md`)
- Horizon: 편집기에서 색이 색으로 보이고, 바꿔진다 (`plans/horizons/2026-07-editor-color-and-token-editing.md`)
- 관측일: 2026-07-21
- 상태: **1차 관측 완료 — 과업 1/3 성공, 2건 미실시.** 재관측(step-3)에서 이어 적는다.

## 이 문서가 다른 증거와 다른 점

ECT1~ECT4의 증거는 내가 만든 것을 내가 잰 것이다. **이 게이트의 증거는 사용자의 행동이다.**
직전 horizon의 `evidence/editor-legibility/eu5-judgeability.md`가 이 문서의 형식 선례다.

## 관측 규약 (준수 확인)

1. **사전 설명 없음** — 지켰다. 과업만 제시하고 방법은 말하지 않았다.
2. 과업은 horizon 닫는 기준 6이 지정한 3건 그대로.
3. 기록은 성공/실패 + 막힌 지점.
4. 막힌 지점은 **관측 중에 고치지 않는다.**
5. 실패를 성공으로 재해석하지 않는다.
6. ⚠ **규약 6은 이번에 새로 붙었다** — "스크린샷 제시로 대체하지 않는다"(직전 horizon에서 실패한 방식). 지켰다: 사용자가 실제 브라우저에서 조작했다.

## 관측 상태

- 문서: 편집기 기본 워크스페이스(`Untitled workspace`) — **1,000노드 개발 fixture**. ⚠ 이 선택이 과업 ②를 막았다(아래 finding 2).
- 서버: `127.0.0.1:5174` (5173이 점유 중이라 vite가 자동 이동)
- 토큰 세트: `편집기 · 밝게`
- 시작: 선택 없음

## 결과 (사용자 수행)

| # | 과업 | 결과 | 막힌 지점 |
|---|---|---|---|
| ① | 색 토큰을 바꾼다 | **성공** | — (안내 없이 찾음) |
| ② | 색이 안 묶인 노드에 색을 묶는다 | **미실시** | 과업 문장의 어휘를 이해 못 함 + 화면에 진입점 없음 |
| ③ | 원시 색으로 벗어난다 | **미실시** | ②에서 관측 중단 |

### 과업 ① — 성공

사용자 원문 그대로:

> *"이게 내 화면이고, 우측 네비게이션에서 배경 색이라는 걸 바꾸니까 네모 하나의 색이 바뀌더라고"*

경로: 우측 속성 패널 → **시각** → **배경 색**. 사전 안내 없이 도달했다.

**이것이 직전 horizon 실패의 정확한 역이다.** EU5에서 사용자는 같은 과업에 *"색은 어떻게
바꾸는지 모르겠네"* 라고 답했다. 그때 화면에 "색"이라는 단어가 0건, 견본이 0개였다.

### 과업 ② — 미실시 (어휘 + 진입점)

사용자 원문 그대로:

> *"무슨 말을 하는지를 모르겠어. 색을 묶으라는게 무슨 말이지? 색을 붙인다는 건 뭐고?"*

그리고 재설명 후에도:

> *"작업 방식이 잘못된거 같아. 너가 어딜 말하는지, 뭘 원하는지 모르겠어"*

여기서 관측을 중단했다.

## 관측이 낸 결함

### finding 1 — 어휘가 안 통한다 (주 결함, ECT5 step-2에서 마감)

사용자가 이해하지 못한 "묶다 / 풀다 / 벗어나다"는 **에이전트가 지어낸 설명이 아니다.**
화면에 그대로 박혀 있는 문구다:

| 위치 | 화면 문자열 |
|---|---|
| `PropertyInspector.tsx:358` | `토큰에서 풀기` |
| `PropertyInspector.tsx:406` | `배경 색 묶기` |
| `PropertyInspector.tsx:478` | `토큰에서 벗어난 색이다. 견본을 눌러 다시 묶을 수 있다.` |
| `PropertyInspector.tsx:464` (aria) | `— 토큰에서 벗어남, 눌러서 다시 묶기` |

즉 **과업 설명의 문제가 아니라 제품의 문제**다. 이 어휘는 ECT3의 milestone 제목("묶고 푼다")
에서 코드로, 코드에서 화면으로 그대로 흘러내렸다 — **내부 설계 언어가 사용자 화면까지
검문 없이 도달한 경로**다.

### finding 2 — 과업 ②의 진입점이 관측 환경에 없다

캔버스 1,000노드를 전수 조사했다.

| 계측 | 값 |
|---|---|
| 노드 총수 | 1,000 |
| 배경색 `oklch(1 0 0)` (`surface.raised`) | 260 |
| 배경색 `oklch(0.95 0.015 270)` (`surface.muted`) | 257 |
| **색이 안 묶인 노드** | **0개** |

사용자가 아무리 찾아도 나올 수 없었다. 다만 **기능이 없는 게 아니다** — `Insert ▸ Frame`으로
새 요소를 넣으면 즉시 `배경 색 묶기` 버튼이 뜬다(실행 확인: `unbound-color-field-background`).
없는 것은 기능이 아니라 **씨앗 fixture 안의 진입점**이다.

→ 이 finding은 ECT5 범위 밖이다. ROADMAP 유지보수 후보
**"프로덕션 셸 정리: 기본 1,000-node fixture·dev 계기판 제거"(QA2 dogfooding 결함 #4)**
와 같은 뿌리이므로 그쪽으로 넘긴다.

### finding 3 — 관측 진행 방식이 틀렸다 (에이전트 측 절차 결함)

에이전트가 **화면을 못 보는 상태로** "설명 없이 해보세요"만 반복해, 어디를 말하는지도 못 짚으면서
판정을 사용자에게 떠넘겼다. 사용자가 이를 직접 지적한 뒤에야 에이전트가 Playwright로 브라우저를
직접 구동하는 방식으로 전환했고, **전환 즉시** 나머지 항목이 픽셀 대조로 판정됐다.

교훈: 사람 관측 게이트에서 **사람에게 물어야 할 것은 "알아볼 수 있는가" 하나뿐**이다.
"색이 정확히 칠해지는가", "detach가 동작하는가"는 에이전트가 재야 하고, 실제로 잴 수 있었다.
이 구분을 미리 안 해서 사용자에게 계측 노동까지 넘겼다.

## 실측으로 기각된 가설

### 가설 A — "견본이 비어 보인다 = 색이 색으로 안 보인다" → **기각**

사용자 화면에서 `surface.muted` 왼쪽 견본이 빈 흰 칸처럼 보였다. 결함을 의심하고 쟀다.

| 대상 | 값 |
|---|---|
| 캔버스 노드 실제 배경 (`getComputedStyle`) | `oklch(0.95 0.015 270)` |
| 인스펙터 견본 배경 | `oklch(0.95 0.015 270)` |
| 견본 `data-resolved` (토큰 해석값) | `oklch(0.95 0.015 270)` |
| 인스펙터 패널 배경 | `oklch(0.16 0.015 270)` |

**셋이 정확히 일치한다.** `surface.muted`가 원래 명도 0.95의 near-white라, 어두운 패널
(명도 0.16) 위에서 빈 칸처럼 읽힌 것이다. 색 오염도, 렌더 누락도 아니다.
→ **닫는 기준 2(색 견본)는 충족된다.**

부수 관측(결함 아님): 밝은 토큰은 어두운 패널에서 "비어 있음"으로 오독될 수 있다.

### 가설 B — "색 없는 요소가 존재하지 않는다" → **기각**

씨앗 fixture에만 없다. `Insert ▸ Frame` 경로로 즉시 만들어진다(위 finding 2).

> ⚠ 이 가설은 **에이전트가 DOM을 읽어서 내린 실사였고, 실행해보니 틀렸다.**
> 이 horizon에서 읽어서 내린 실사가 틀린 **4번째** 사례다
> (① 어휘 이름 겹침 ② WebGPU 색 오염 ③ 이미지 렌더 ④ 이 건).
> **실행해서 재는 규율이 다시 옳았다.**

## 에이전트 구동으로 확인한 항목 (사람 관측 아님 — 구분해 기록)

과업 ③은 사람이 수행하지 못했으나, 기능 자체는 에이전트가 브라우저에서 확인했다.

- `토큰에서 풀기` 클릭 → 필드가 `color-field-background` → `literal-color-field-background`로 전환.
- 토큰 이름 자리가 **직접 입력 칸**(`literal-input-background`)으로 바뀌고 값 `oklch(1 0 0)` 표시.
- 안내문 `토큰에서 벗어난 색이다. 견본을 눌러 다시 묶을 수 있다.` 출력 — **되돌아갈 길이 화면에 있다.**
- 견본은 `color-swatch-literal` 클래스가 붙어 형태로도 구분된다(리서치 1의 규약 준수).

**동작은 정상이다. 못 넘은 것은 어휘다.**
