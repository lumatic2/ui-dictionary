# 랜딩/마케팅 페이지 구성(섹션 조합·순서) 패턴 카탈로그

> 조사일: 2026-07-19. 목적: 디자인 브리프 도구의 "페이지 구성" 선택지가 "히어로+카드3+정보" 단조 구성에 머무는 문제 해소.

## 1. 요약

- 구성 유형 **12종**을 카탈로그화했다 (요청 목표 10종 이상 충족).
- 공통 원리: 모든 유형이 결국 "①첫 화면에서 무엇을 증명할지 → ②그다음 저항(objection)을 어떤 순서로 해소할지 → ③행동을 어디서 반복할지"를 다르게 배열한 변주다. 즉 섹션 종류(히어로/기능/증거/CTA) 자체는 크게 다르지 않고, **순서·비중·시각 밀도**가 유형을 가른다.
- 주목할 패턴 3가지:
  1. **증거(social proof)의 위치가 유형을 가르는 축이다** — 표준형은 증거를 3번째 섹션에 몰아넣지만, 회의적 시장을 겨냥한 유형(FAQ/신뢰형, 비교표형)은 증거를 히어로 직후 또는 각 반박 지점마다 분산 배치한다.
  2. **원스크린형과 롱폼 스토리텔링형은 정반대 극단**이지만 둘 다 "단일 메시지 집중"이라는 같은 원리에서 나온다 — 전자는 압축, 후자는 시간축으로 펼친 압축이다.
  3. **예약/티켓형은 일반 랜딩 구성 원리를 그대로 쓰되 위젯(검색/좌석/캘린더)이 히어로 안에 통합되고, 긴급 신호(잔여 수량·카운트다운)가 CTA 옆에 항상 붙는다**는 점에서 구조적으로 별도 카테고리다.

## 2. 구성 유형 카탈로그

### 2.1 표준 퍼널형 (Hero → Value → Social Proof → CTA)
- **섹션 시퀀스**: 히어로(헤드라인+서브헤드+비주얼+CTA) → 가치제안(3~4개 요약, 아이콘/3열 그리드) → 로고 스트립/사회적 증거 → 기능 상세 → 2차 CTA → FAQ → 최종 CTA.
- **적합 맥락**: 범용 SaaS/제품 랜딩, 첫 방문자 다수, 브랜드 인지도 낮음.
- **출처**: [Replo — Anatomy Of A Landing Page](https://www.replo.app/blog/anatomy-of-a-landing-page) (접속 2026-07-19), [Unicorn Platform — Landing Page Structure in 2026](https://unicornplatform.com/blog/optimal-landing-page-structure/) (접속 2026-07-19)

### 2.2 Z-패턴 스플릿 히어로형
- **섹션 시퀀스**: 스플릿 히어로(좌 텍스트/우 이미지 또는 반대, 대각선 시선 유도) → 신뢰 로고 바 → 기능 3분할 → 단일 강한 CTA. 짧은 리드젠 페이지에 적합.
- **적합 맥락**: 단일 메시지·단일 CTA가 목표인 광고 랜딩, 인증/온보딩 페이지, 가격 비교 페이지.
- **출처**: [Orizon Design — Common Hero Section Design Layouts](https://www.orizon.co/blog/common-hero-section-design-layouts-and-when-to-use-them) (접속 2026-07-19), [Qode Interactive — Split Screen Layout in Use](https://qodeinteractive.com/magazine/split-screen-layout-in-use-best-examples/) (접속 2026-07-19)

### 2.3 풀블리드 미디어(시네마틱)형
- **섹션 시퀀스**: 전체 화면 배경 영상/이미지 히어로(자동재생 루프, 하단 마스크 그라디언트로 다음 섹션과 연결) → 짧은 카피 오버레이 → 제품 디테일 클로즈업 시퀀스 → 최소한의 텍스트 CTA.
- **적합 맥락**: 럭셔리 브랜드, 제품 런칭, 부동산/여행 등 분위기·무드가 구매 결정에 큰 비중을 차지하는 업종.
- **출처**: [sitesplaced — Cinematic Landing Pages with Video Backgrounds](https://sitesplaced.com/blog/cinematic-landing-pages-with-video-backgrounds) (접속 2026-07-19)

### 2.4 스텝 온보딩형 ("How it works")
- **섹션 시퀀스**: 히어로 → 3~4단계 수직/수평 스텝 블록(각 스텝에 아이콘+짧은 설명, 2톤 배경으로 구간 구분) → 스텝별 스크린샷/데모 → 완료 후 결과 강조 → FAQ → CTA.
- **적합 맥락**: 프로세스가 명확한 제품(앱, 플랫폼, 서비스형 워크플로우), 신규 사용자의 "어떻게 쓰는지" 불확실성이 이탈 원인일 때.
- **출처**: [land-book — How it works section inspiration](https://land-book.com/sections/how-it-works) (접속 2026-07-19), [SaaS Landing Page — 10+ Brilliant How it Works Page Examples](https://saaslandingpage.com/articles/10-brilliant-how-it-works-page-examples/) (접속 2026-07-19)

### 2.5 소셜프루프 선행형
- **섹션 시퀀스**: 히어로 직후(또는 히어로 내부) 로고 스트립/별점/고객 영상 → 문제 정의 → 해결책 → 각 기능 옆에 개별 고객 인용구 분산 배치 → 최종 CTA. 증거를 한 블록에 몰지 않고 저항 지점마다 재배치하는 것이 핵심.
- **적합 맥락**: 회의적인 B2B 구매자, 고관여 결정, 경쟁사 대비 신뢰 부족을 극복해야 하는 신생 브랜드.
- **출처**: [SaaSHero — Landing Page Social Proof: 12 Tactics](https://www.saashero.net/content/landing-page-social-proof-examples/) (접속 2026-07-19)

### 2.6 롱폼 스토리텔링(스크롤리텔링)형
- **섹션 시퀀스**: 도입 훅 → 시간축/서사 순서로 전개되는 연속 섹션(스크롤 트리거 애니메이션, 스티키 배경 전환, 하나의 씬당 하나의 통찰) → 클라이맥스(핵심 증거/전환점) → 결말 CTA. 전통적 섹션 구분보다 "장면(scene)" 단위로 설계.
- **적합 맥락**: 브랜드 스토리, 임팩트 리포트, 복잡한 개념을 단계적으로 설득해야 하는 캠페인 페이지.
- **출처**: [Shorthand — Scrollytelling examples](https://shorthand.com/the-craft/scrollytelling-examples/index.html) (접속 2026-07-19), [Flourish — What great scrollytelling looks like](https://flourish.studio/blog/scrollytelling-examples/) (접속 2026-07-19)

### 2.7 비교표(Us vs Them)형
- **섹션 시퀀스**: 히어로(경쟁사 대비 포지셔닝 명시) → 체크/크로스 아이콘 비교 테이블(자사 vs 경쟁사 vs 기존 방식) → 가격 비교 → 전환 고객 사례("~에서 이전한 이유") → CTA. 구매 여정 후반부(고관여) 트래픽 전용 구성.
- **적합 맥락**: 경쟁이 명확한 SaaS 카테고리, 이미 대안을 알아본 고의도 트래픽.
- **출처**: [Reviewflowz — 15 Competitive Comparison Landing Page Examples](https://www.reviewflowz.com/blog/competitive-comparison-landing-page-examples) (접속 2026-07-19), [SaaS Landing Page — 15 Best Comparison Page Examples](https://saaslandingpage.com/articles/15-best-comparison-page-examples-and-why-they-work/) (접속 2026-07-19)

### 2.8 갤러리/쇼케이스(비주얼 우선)형
- **섹션 시퀀스**: 미니멀 헤더 → 매소너리/그리드 갤러리(작업물 나열, 텍스트 최소화) → 선택 시 상세 케이스로 연결 → 하단에만 짧은 소개/CTA. 텍스트 설득보다 작업물 자체가 증거 역할.
- **적합 맥락**: 포트폴리오, 에이전시, 디자인/사진/건축 등 결과물이 스스로 말하는 업종.
- **출처**: [Visual Portfolio — Choosing the Right Layout for Your Visual Portfolio Gallery](https://www.visualportfolio.com/docs/gallery-blocks/layout/) (접속 2026-07-19)

### 2.9 원스크린(No-Scroll)형
- **섹션 시퀀스**: 뷰포트 하나에 헤드라인+서브카피+단일 CTA(또는 단일 입력창)만 배치. 별도 섹션 스크롤 없음. 기능/증거는 생략하거나 클릭 후 별도 페이지로 위임.
- **적합 맥락**: 콜드 광고 트래픽, 브랜드 인지도가 이미 높은 제품(Linear, Vercel류), 단일 액션(검색·가입)이 유일 목표.
- **출처**: [Landdding — 15 Minimal Landing Page Design Examples](https://landdding.com/blog/minimal-landing-page-design-examples) (접속 2026-07-19)

### 2.10 FAQ·신뢰형 (고관여/고가 서비스)
- **섹션 시퀀스**: 히어로 → 문제/저항 지점 명명 → 프로세스 상세(고가 서비스는 "무슨 일이 일어나는지" 설명이 김) → 저항별로 매핑된 개별 증거(사례·리뷰) → 아코디언 FAQ(감정적 반론까지 포함) → 보증/환불 정책 → CTA. 순서가 "회사가 말하고 싶은 것"이 아니라 "구매자가 걱정하는 순서"를 따른다.
- **적합 맥락**: 컨설팅, 하이엔드 코칭, B2B 엔터프라이즈, 의료/법률처럼 신뢰가 전환의 병목인 서비스.
- **출처**: [Unicorn Platform — Landing Page Structure in 2026](https://unicornplatform.com/blog/optimal-landing-page-structure/) (접속 2026-07-19)

### 2.11 문제-메커니즘-증거형 (Problem → Mechanism → Proof)
- **섹션 시퀀스**: 문제 정의(공감 유도) → 왜 기존 해법이 실패하는지 → 독자적 메커니즘/차별점 설명 → 메커니즘이 작동한다는 증거(데이터/케이스) → 반론 처리 → CTA. 기능 나열형과 달리 "왜 이게 다른가"를 논증 구조로 배치.
- **적합 맥락**: 카테고리 창조형 스타트업, 기존 통념을 뒤집어야 하는 제품, 교육적 세일즈가 필요한 고관여 B2B.
- **출처**: [SaaSHero — Landing Page Social Proof](https://www.saashero.net/content/landing-page-social-proof-examples/) (접속 2026-07-19) — "objection module"·"mechanism" 섹션 근거

### 2.12 커뮤니티/UGC 우선형
- **섹션 시퀀스**: 히어로 → 실사용자 콘텐츠 그리드(리뷰·SNS 임베드·커뮤니티 활동) → 제품 기능은 UGC 사이 사이에 소량만 삽입 → 커뮤니티 참여 CTA(가입/다운로드). 브랜드가 아니라 사용자가 화자.
- **적합 맥락**: 소비자 앱, 크리에이터 툴, 커뮤니티 기반 제품(예: 피트니스·취미 플랫폼).
- **출처**: [Wisernotify — 10 High-Impact Social Proof Landing Page Examples](https://wisernotify.com/blog/landing-page-social-proof/) (접속 2026-07-19)

## 3. 예약/티켓/이커머스형 특수 패턴

일반 퍼널 구성 원리를 공유하지만 아래 3가지가 구조적으로 다르다.

1. **검색/예약 위젯이 히어로 안에 통합된다** — 별도 섹션이 아니라 히어로 첫 화면에 날짜·인원·목적지 입력 위젯이 바로 노출된다 (호텔: 체크인/체크아웃 캘린더, 레스토랑: 예약 CTA가 헤더에 고정, 이벤트: 티켓 유형 선택이 히어로 하단에 붙음).
2. **잔여·긴급 신호가 CTA 옆에 상시 배치된다** — "Only 3 rooms left", "Only 25 spots left!" 같은 스캐어시티 카피, 카운트다운 타이머는 푸터가 아니라 CTA 바로 옆(above the fold)에 위치해야 결정 시점에 압박이 작동한다.
3. **단계 안내가 좌석/객실 단위로 시각화된다** — 이벤트는 좌석 맵(좌석별 티켓 등급 연결), 호텔은 객실 타입별 가용 캘린더, 레스토랑은 시간대별 예약 슬롯. 일반 SaaS의 추상적 "3단계 프로세스" 대신 실제 재고/공간을 시각적으로 매핑한다.

- **출처**: [Booking Factory — 5 Urgency tactics to increase hotel website conversions](https://www.bookingfactory.io/blog/5-urgency-tactics-to-increase-hotel-website-conversions) (접속 2026-07-19), [Fanz — Event Landing Page: Structure That Sells Tickets Fast](https://fanz.com.ar/en/blog/event-landing-page-structure-that-sells-tickets) (접속 2026-07-19), [Tickera — Seating Charts That Reduce Confusion and Refunds](https://blog.tickera.com/seating-chart-best-practices/) (접속 2026-07-19), [ElevateWebDesign — Restaurant Website Design Examples](https://elevatewebdesign.ca/blog/restaurant-website-design-examples) (접속 2026-07-19)

> 참고: Booking.com류 "room left" 경고의 상당수가 실제 재고를 반영하지 않는다는 비판이 있다 — [NBC News — The travel website you're using says there's 'only 1 room left' — is that true?](https://www.nbcnews.com/better/lifestyle/travel-website-you-re-using-says-there-s-only-1-ncna1073066) (접속 2026-07-19). 긴급 신호 패턴을 브리프 도구에 반영할 때 "실재하는 재고"를 전제로 명시할 필요.
