# 생성형 템플릿 제작 시스템 역공학

Date: 2026-07-19  
Consumer: 제안 Horizon `template-production-system`의 설계 토론 입력  
Status: research complete — 공개 제품·문서·클라이언트 자산 기준, 내부 구현은 추정으로 분리

## 질문

미리캔버스에서 보인 명함·포스터·인포그래픽 같은 결과는 GPT Image 2 프롬프트만으로 만들어지는가, 아니면 별도 템플릿·레이아웃·편집 시스템이 결합되는가? Askewly Design이 같은 능력을 갖추려면 무엇을 정본으로 가져야 하는가?

## 한 줄 결론

**GPT Image 2는 강력한 시각 소재·완성 이미지 생성기지만, 편집 가능한 템플릿 시스템 자체는 아니다.** 공개 증거가 가리키는 실전 구조는 `요구 해석 → 콘텐츠 구조화 → 용도별 구성 선택/생성 → 토큰·브랜드 적용 → 이미지 생성 → 편집 가능한 장면 조립 → 후보 비교·수정·내보내기`의 혼합 파이프라인이다.

## 1. 미리캔버스에서 확인된 것

### 1.1 템플릿·요소 라이브러리가 먼저 존재한다

- 공식 홈은 53만 개 이상의 템플릿과 1,000만 개 이상의 사진·아이콘·영상·오디오 요소를 별도 자산층으로 제시한다.
- 템플릿은 프레젠테이션, 카드뉴스, 상세페이지, 로고, 명함, 포스터, 전단지, 현수막 등 **용도와 출력 규격**으로 분류된다.
- 따라서 AI가 매 요청마다 빈 화면에서 모든 구성 규칙을 발명한다고 보기 어렵다. 이미 큰 템플릿·요소 검색 공간이 있다.

Source: [미리캔버스 공식 홈](https://www.miricanvas.com/ko) (accessed 2026-07-19)

### 1.2 일반 AI 이미지 생성과 템플릿 제작은 분리돼 있다

- 공식 AI 이미지 페이지는 FLUX, DALL-E 3, Stable Diffusion 등을 이미지 생성 모델로 열거한다.
- 흐름은 `유형/스타일 선택 → 프롬프트·참고 이미지 → 이미지 생성 → 생성 이미지를 미리캔버스 템플릿에 활용`이다.
- 생성 결과는 에디터에서 필터·효과·색상 조정과 텍스트·그래픽 추가가 가능하지만, 공식 설명은 생성 이미지를 자동으로 편집 가능한 텍스트·도형 계층으로 만든다고 말하지 않는다.

Source: [미리캔버스 AI 이미지 생성](https://www.miricanvas.com/ko/features/image-generator) (accessed 2026-07-19)

### 1.3 AI 프레젠테이션은 명백한 혼합 파이프라인이다

공식 사용 흐름은 다음과 같다.

1. 주제 또는 PDF 입력
2. 목적·청중에 맞는 개요 생성 및 사용자 수정
3. 발표 목적과 분위기에 맞는 디자인 테마 선택
4. 선택한 테마의 색상·폰트·레이아웃 균형 적용
5. 페이지 맥락에 맞는 이미지 삽입
6. 텍스트·이미지·레이아웃을 요소별 편집하고 PPTX 등으로 내보내기

이 흐름은 하나의 이미지 프롬프트가 아니라 **내용 모델 + 테마/템플릿 + 구조화된 편집 문서 + 이미지 도구**의 결합이다.

Source: [미리캔버스 AI 프레젠테이션](https://www.miricanvas.com/ko/features/ai-presentation-maker) (accessed 2026-07-19)

### 1.4 AI 포스터 공개 클라이언트에서 관찰한 단서

공개 웹 자산을 읽기 전용으로 확인했다. 인증 우회·비공개 API 호출은 하지 않았다.

- 공개 번들은 `ai-poster`를 `image-generate`, `ai-logo`, `ai-series`, `redesign`, `image-to-asset` 등과 나란한 생성 작업 유형으로 열거한다.
- 공통 입력 모델에는 텍스트, 참고 이미지 목록, 종횡비, 프리셋, 프리셋의 모델 라벨이 있다.
- 작업 결과의 공개 클라이언트 표현은 `IMAGE`, `VIDEO`, `AUDIO` 유형이며, 포스터 화면은 공통 이미지 생성 활동·다운로드 흐름을 재사용한다.
- 한국어 문자열은 포스터 생성 단계를 `아이디어 이해 → 어울리는 디자인 탐색 → 구성 생성 → 한글 표시 확인 → 포스터 그리기 → 마무리`로 표현한다.

**추론:** 현재 공개 AI 포스터 표면은 “편집 가능한 장면을 처음부터 생성”하기보다, 구성과 텍스트를 포함한 포스터 이미지를 생성하고 결과를 저장/활용하는 **래스터 우선 경로**일 가능성이 높다. 다만 로그인 후 에디터 전달 과정이 생성 결과를 요소로 재구성하는지는 공개 자산만으로 확정할 수 없다.

### 1.5 GPT Image 2 사용 여부

- 현장 화면에서 GPT Image 2 선택을 보았다는 사용자 관찰은 중요한 1차 단서다.
- 그러나 미리캔버스의 공개 AI 이미지 페이지는 GPT Image 2를 명시하지 않고, 공개 포스터 번들도 프리셋의 실제 모델 매핑을 노출하지 않는다.
- 그러므로 **미리캔버스가 어떤 기능에 GPT Image 2를 쓰는지는 현재 공개 근거만으로 확정하지 않는다.** 계정·지역·실험군·행사 버전 또는 비공개 프리셋 차이일 수 있다.

## 2. GPT Image 2가 담당할 수 있는 것과 없는 것

### 담당하기 좋은 것

- 텍스트에서 새 이미지 생성, 기존 이미지의 정밀 편집, 여러 참고 이미지 합성
- 마케팅 크리에이티브, 인포그래픽, UI 모형, 제품 목업, 실제 문구가 들어간 이미지의 초안
- 임의 종횡비 범위의 출력, 품질·포맷·압축 제어, 대화형 반복 편집
- 같은 구도·제품·인물·브랜드 소재를 유지하면서 배경·조명·장면 변형

### 단독으로 맡기면 안 되는 것

- 텍스트 상자, 도형, 차트, 이미지 프레임, 그룹, 제약 조건이 살아 있는 편집 장면
- 명함의 재단선·안전 여백, 인포그래픽의 데이터 정확성, 포스터의 실제 한글 폰트 파일 같은 구조적 계약
- 브랜드 토큰을 모든 요소에 결정론적으로 적용하는 일
- 같은 콘텐츠를 여러 규격으로 변환할 때 요소 관계를 보존하는 일
- 결과를 다시 생성하지 않고 개별 텍스트·아이콘·차트를 수정하는 일

OpenAI 공식 문서는 GPT Image 2를 생성·편집 모델로 설명하며, 결과는 이미지 데이터다. 프롬프트 가이드도 인포그래픽·UI 모형·마케팅 크리에이티브를 다루지만, 통제 수단은 명시적 제약과 반복 편집이다. **편집 가능한 디자인 장면은 제품이 별도로 소유해야 한다.**

## 3. 유사 서비스가 보여주는 구조

### Canva: 디자인 모델 + 편집 장면 + 래스터 분해

- Magic Design은 이미지/프롬프트와 스타일을 받아 폰트·그래픽·스타일이 조합된 후보 템플릿을 큐레이션한다.
- Canva는 자체 Design Model이 처음부터 편집 가능한 프레젠테이션·문서·소셜 게시물을 생성한다고 설명한다.
- Magic Layers는 별도 경로다. 평면 이미지를 해석해 텍스트, 객체, 배경, 레이아웃 관계를 복원하고 편집 가능한 계층으로 재구성한다.

이 사례는 `구조를 처음부터 생성하는 경로`와 `평면 이미지를 나중에 분해하는 경로`가 서로 다른 문제임을 명확히 보여준다.

Sources:

- [Canva Magic Design 소개](https://www.canva.com/newsroom/news/supercharging-the-visual-suite/) (accessed 2026-07-19)
- [Canva Magic Layers 소개](https://www.canva.com/newsroom/news/magic-layers/) (accessed 2026-07-19)

### Adobe Express: 이미지 모델이 아니라 Design Model로 편집 템플릿 생성

- Text to Template은 설명에서 소셜 게시물·전단지·포스터·카드를 만들고, 여러 후보 중 하나를 골라 텍스트·브랜드·이미지를 편집한다.
- Adobe는 이 기능을 Firefly Design Model 기반의 편집 가능한 템플릿 생성으로 설명한다.

Sources:

- [Adobe Express 편집 가능한 템플릿 생성 안내](https://helpx.adobe.com/express/web/create-with-templates/text-to-template.html) (accessed 2026-07-19)
- [Adobe Express AI Template Generator](https://www.adobe.com/express/create/ai/template-generator) (accessed 2026-07-19)

### Piktochart: 용도별 구조 선택 + 콘텐츠 구조화 + 기존 편집기

- 사용자는 인포그래픽 아래에서 목록·비교·타임라인·차트·뉴스레터 같은 하위 형식을 먼저 고른다.
- 프롬프트, 붙여넣은 글, 문서에서 내용을 구조화하고 여러 시각 후보를 만든 뒤 하나를 선택한다.
- 공식 도움말은 생성 단계에서 레이아웃 지시를 따르지 않으며, 레이아웃·구조는 생성 후 편집기에서 조정하라고 명시한다.
- 문서 속 차트는 네이티브 편집 차트로 보존하는 별도 변환 기능을 둔다.

## 4. 역공학된 공통 시스템

```text
사용자 요구
  → 용도·규격 판정
  → 콘텐츠 구조화
  → 구성 청사진 후보
  → 시각 토큰·브랜드 규칙
  → 요소 계획
  → 이미지·아이콘 소재 생성/검색
  → 편집 가능한 장면 조립
  → 후보 비교·선택
  → 직접 편집·규격 변환·내보내기
  → 품질·정확성 검증
```

각 층의 책임은 다음처럼 나뉜다.

| 층 | 정본 | 생성 모델의 역할 |
|---|---|---|
| 용도·규격 | 명함/포스터/인포그래픽별 계약 | 요구를 분류하고 빈 필드 질문 |
| 콘텐츠 구조 | 제목·본문·통계·CTA·연락처·출처 | 원문 요약·계층화·카피 후보 |
| 구성 청사진 | 슬롯·그리드·여백·우선순위·반응 규칙 | 후보 선택·매개변수 제안 |
| 시각 체계 | 토큰·폰트·색·형태·브랜드 제약 | 분위기 해석·조합 추천 |
| 소재 | 사진·일러스트·아이콘·배경 | GPT Image 2 생성·편집 또는 스톡 검색 |
| 편집 장면 | 요소 트리·그룹·제약·데이터 연결 | 장면 명령 제안, 결과 자체는 구조화 |
| 검증 | 텍스트 정확성·충돌·가독성·출력 규격 | 비평·수정 제안, 기계 검증과 병행 |

## 5. Askewly Design 편입안

### 현재 자산과 맞닿는 지점

- `design-brief.md`: 정체성·톤·키워드·성공 기준을 수집한다.
- `brief-studio.md`: 스타일 타일과 18개 시각 축을 실물로 고르고 `DESIGN.md`로 저장한다.
- `studio-data.default.json`: 선택 후보와 추천 순서를 데이터로 주입한다.
- `make-studio.py`: 데이터 → 자기완결 HTML이라는 작은 컴파일 경계를 이미 증명했다.
- Architecture의 canonical canvas document: 프레임·그룹·텍스트·레이아웃 제약·토큰 바인딩을 담을 미래 편집 장면과 직접 연결된다.

현재 시스템에는 **룩을 고르는 단계**는 있지만, 특정 산출물의 콘텐츠를 구조화하고 편집 장면으로 조립하는 단계가 없다.

### 추천: 구조 우선 경로

첫 구현은 평면 이미지를 만들어 다시 분해하지 않는다.

1. 명함·포스터·인포그래픽별 입력 계약을 만든다.
2. 콘텐츠를 타입 있는 블록으로 구조화한다.
3. 용도별 구성 청사진에서 슬롯과 제약을 선택한다.
4. Brief Studio의 선택과 `DESIGN.md` 토큰을 적용한다.
5. GPT Image 2는 배경·주제 이미지·일러스트·장식 소재에만 사용한다.
6. 결과를 HTML/CSS/SVG 또는 향후 canonical canvas document의 편집 가능한 요소로 조립한다.
7. 래스터 참고안은 후보 생성·비평용으로만 병행한다.

이 경로는 Askewly Design의 강점인 **에이전트가 읽는 규칙, 실제 코드, 검증, 무손실 왕복**과 맞는다.

### 나중에 검토할 경로: 평면 이미지 → 편집 장면 복원

Canva Magic Layers와 같은 분해는 별도 연구 과제다. OCR, 객체 분리, 배경 복원, 폰트 추정, 요소 간 제약 복구, 원본과의 시각 차이 평가가 모두 필요하다. 첫 버전에 섞으면 제품의 정본이 다시 픽셀로 퇴행한다.

## 6. 먼저 합의할 제품 결정

1. **첫 산출물 범위**  
   추천: `명함 + 제품 포스터 + 인포그래픽` 3종. 서로 다른 구조 문제(고정 인쇄 규격 / 마케팅 위계 / 정보 구조)를 작게 대표한다.
2. **초기 편집 표면**  
   추천: 브라우저의 HTML/CSS/SVG 편집 미리보기 + JSON 장면. 데스크톱 canvas 정본에는 계약이 안정된 뒤 연결한다.
3. **GPT Image 2의 위치**  
   추천: 소재 생성기 + 래스터 참고안. 전체 템플릿을 유일한 정본으로 생성하지 않는다.
4. **후보 생성 방식**  
   추천: 용도별 구성 청사진 4~8개를 규칙 기반으로 조립하고, 모델은 선택·매개변수·카피·소재를 제안한다.
5. **편집성 기준**  
   추천: 텍스트·이미지·도형·차트·그룹·토큰 바인딩이 독립 편집돼야 “템플릿”으로 인정한다. 평면 이미지는 “시안”으로 부른다.
6. **외부 모델·비용**  
   추천: 첫 구조 검증은 생성 모델 없이 고정 fixture로 닫고, GPT Image 2 실제 호출은 별도 비용·키 승인 뒤 추가한다.

## 7. 리스크와 검증 질문

- 예쁜 래스터 한 장을 성공으로 착각하면 Askewly Design의 코드·편집 정본 목표와 충돌한다.
- 인포그래픽의 수치·출처를 이미지 모델에 맡기면 사실 정확성을 검증할 수 없다.
- 한글 텍스트를 이미지 안에 굽는 방식은 수정·접근성·검색·PPTX/SVG 내보내기를 약화한다.
- 구성 청사진을 너무 적게 두면 결과가 반복되고, 너무 많이 두면 무작위 템플릿 검색으로 퇴행한다.
- Brief Studio의 시각 축과 산출물 구성 청사진을 한 JSON에 섞으면 “룩 선택”과 “문서 구조”의 책임이 흐려진다.
- 실제 GPT Image 2 호출 전에 고정 입력·고정 장면으로 결정론적 렌더와 편집성을 먼저 검증해야 한다.
