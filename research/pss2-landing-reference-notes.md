## Transferable Patterns

- Tailwind: make the page itself demonstrate the system. Minimal decoration works when the code/result example is strong.
- Linear: use one dominant product surface as proof instead of equal-weight feature cards.
- shadcn/ui: a design-system landing can be a component/specimen collage.
- Framer: agent input plus generated result creates a stronger AI-era narrative than generic CTA copy.
- Vercel: strong whitespace and iconography can create authority, but Askewly Design still needs visible UI proof before it can go that minimal.

## PSS2 Direction

The homepage should feel like an interface atlas, not a two-column SaaS landing. The first viewport now uses:

- oversized Askewly Design typography;
- a prompt bar that frames the agent workflow;
- overlapping UI specimens for SaaS, mobile, commerce, docs, tokens, and Pro code;
- compact entry tiles for Docs, Patterns, Showcase, Resources, and Pro.
