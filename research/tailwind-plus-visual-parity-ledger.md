# Tailwind Plus visual parity ledger

Date started: 2026-07-01
Purpose: track leaf-by-leaf visual fidelity work after route/capture parity.

This ledger is stricter than `docs/research/tailwind-plus-capture-ledger.md`. A row here is not visually complete just because the local route renders or has screenshots. Visual completion requires fresh Tailwind desktop/mobile capture, screenshot-backed local implementation, post-implementation local desktop/mobile capture, and verification.

## Status Values

- `planned`: row is in scope, but the visual pass has not started.
- `captured`: fresh Tailwind reference capture or baseline evidence exists, but local visual implementation is not complete.
- `implemented`: local page has been updated from the current reference capture, but verification is still pending.
- `visual-parity-verified`: local page passed the full visual parity gate for this row.
- `deferred`: row is intentionally skipped with a specific public-access or scope reason.
